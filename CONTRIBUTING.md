# Contributing to the Gemini FullStack ADK

We welcome contributions from the community and are excited to see what you build with the Gemini FullStack ADK. By contributing, you can help us improve the documentation and make it even more useful for everyone.

## How to Contribute

There are several ways you can contribute to this project:

*   **Reporting Bugs**: If you find a bug in the documentation, please [open an issue](https://github.com/google/gemini-fullstack-adk/issues) on our GitHub repository.
*   **Suggesting Enhancements**: If you have an idea for a new feature or an improvement to an existing one, please [open an issue](https://github.com/google/gemini-fullstack-adk/issues) to discuss it.
*   **Submitting Pull Requests**: If you've fixed a bug or implemented a new feature, you can submit a pull request.

## Pull Request Process

1.  Fork the repository and create your branch from `main`.
2.  If you've added code that should be tested, add tests.
3.  Ensure your work is well-documented.
4.  Make sure your code lints.
5.  Issue that pull request!

## Style Guides

We follow the [Google Developer Style Guide](https://developers.google.com/style) and the [Microsoft Writing Style Guide](https://docs.microsoft.com/en-us/style-guide/welcome/). Please ensure your contributions adhere to these guides.

## Code of Conduct

We have a [Code of Conduct](./CODE_OF_CONDUCT.md) that all contributors are expected to follow. Please make sure you are familiar with it.

Thank you for your contributions!
